## V2.5 新增功能

### 1. 5层对标过滤系统

在原有竞品分析基础上,增加dbskill的5层过滤标准,用于识别**真正值得对标的竞品**。

#### 5层过滤标准

| 层级 | 标准 | 说明 | 检查方法 |
|------|------|------|----------|
| **L1** | 目标对象利润显著 | 对标对象收益达到你预期的10倍以上 | 分析其变现数据/粉丝量/互动率 |
| **L2** | 你能理解其模式 | 能清晰拆解对方的商业逻辑 | 分析内容结构/发布频率/变现路径 |
| **L3** | 你能复刻执行 | 具备复制所需的基本能力 | 评估技术门槛/资源需求/时间投入 |
| **L4** | 忽略现有资源/偏好 | 不因"我没有XX"或"我不喜欢XX"而放弃 | 客观评估,排除主观偏好 |
| **L5** | 执行而非过度思考 | 不纠结业务类型是否"高级" | 接受难度是过滤器 |

#### 对标对象分级

通过5层过滤后,对标对象分为3个等级:

| 等级 | 通过层级 | 建议 | 对标价值 |
|------|----------|------|----------|
| **S级** | 全部通过 | 像素级复刻 | 极高,立即对标 |
| **A级** | 通过L1-L3 | 部分借鉴 | 高,重点参考 |
| **B级** | 仅通过L1-L2 | 了解即可 | 中,作为参考 |
| **C级** | 仅通过L1 | 不建议对标 | 低,仅供观察 |

## 使用指南

### 场景1: 寻找对标对象

```yaml
# 输入
mode: compete
topic_id: "ai-agent-development-guide"

# 输出
- 识别所有竞品
- 应用5层过滤标准
- 推荐S/A/B/C级对标对象
- 生成复刻计划
```

### 场景2: 验证对标可行性

```yaml
# 输入
mode: compete

- 对指定对象进行5层过滤
- 评估复刻可行性
- 生成详细复刻清单
```

```yaml
- 日历排期
```

## 对标方法论核心原则

### 原则1: 对标优于创新
> "从0到1阶段,找到已经超额完成目标的人,完整复刻其行为模式,从而消解问题本身。"

### 原则2: 像素级复刻
> "不是'差不多',而是可观察维度的完全一致。"

### 原则3: 执行100次
> "至少100次迭代后再评估效果,不要过早放弃。"

### 原则4: 问题消解而非问题解决
> "不要问'我该怎么做',而是'他怎么做的,我就怎么做'。"

## 对标执行清单

### 阶段1: 选择对标对象 (1-2天)
- [ ] 搜索同类内容,收集10-20个竞品
- [ ] 应用5层过滤标准
- [ ] 识别S级对标对象(至少1个)
- [ ] 如果没有S级,重新思考选题

### 阶段2: 拆解对标对象 (3-5天)
- [ ] 分析内容结构和章节安排
- [ ] 记录发布频率和时间
- [ ] 拆解变现路径和产品
- [ ] 研究互动方式和话术
- [ ] 统计关键数据指标

### 阶段3: 制定复刻计划 (1-2天)
- [ ] 列出所有可观察维度
- [ ] 制定像素级复刻清单
- [ ] 设定执行时间表
- [ ] 确定成功指标

### 阶段4: 执行复刻 (12周)
- [ ] 完全按照对标对象执行
- [ ] 不做任何"优化"或"改进"
- [ ] 记录每次执行的数据
- [ ] 对比与对标对象的差异

### 阶段5: 数据对比与调整 (持续)
- [ ] 每周对比关键指标
- [ ] 识别执行偏差
- [ ] 校准至与对标一致
- [ ] 100次后评估是否继续

## 常见问题

**Q: 对标是不是抄袭?**
A: 对标是学习商业模式和执行方法,不是复制具体内容。就像学习烹饪,你学的是配方和技法,不是直接卖别人做的菜。

**Q: 对标对象太成功,我根本做不到怎么办?**
A: 这正是对标的价值——它告诉你目标是可达成的。如果做不到,问题在执行,不在目标。先执行100次再说。

**Q: 我的情况和对标对象不一样怎么办?**
A: 0到1阶段,差异不重要。先证明模式可行,再谈个性化。

**Q: 对标多久才能看到效果?**
A: 至少100次迭代。如果100次后还没效果,检查是否真的做到了"像素级复刻"。

**Q: 可以同时对标多个对象吗?**
A: 不建议。单一对标优于多个对标。选择最成功的那个,完全复刻。

## 配置参数

```yaml
topic_validate:
  version: "2.5"

  # 5层对标过滤
  benchmark_filter:
    enabled: true
    min_grade: "A"  # 最低接受等级
    auto_generate_plan: true  # 自动生成复刻计划

  # 过滤阈值
  filter_thresholds:
    level_1_profit: 7.0
    level_2_understand: 6.0
    level_3_replicate: 6.0
    level_4_objective: 7.0
    level_5_execute: 7.0

  # 对标对象数量
  benchmark_targets:
    s_grade: 1-2  # S级对标对象
    a_grade: 2-3  # A级参考对象
    total_analyze: 10  # 总分析数量
```
