## 技能定位

商业模式诊断技能,在选题阶段验证内容的商业可行性,通过3层诊断框架(商业层、执行层、认知层)识别潜在问题,应用问题消解方法论,确保选题具备商业价值。

**核心价值**:
- 验证商业可行性
- 识别执行障碍
- 消解认知误区
- 提供可执行方案
- 避免无效投入

## 核心公理

> **公理**: 执行力 + 商业模式 = 赚钱

### 公理1: 问题识别的本质
"很多赚不到钱的人其实知道答案,只是在明知正解的情况下竭尽全力寻找绕过它的方法。"

### 公理2: 产品的定义
"如果不能发付款链接让人成功付款,就是没有产品。"

### 公理3: 商业模式的本质
"供需匹配到极致、人货匹配到极致、交易摩擦缩小到极致。"

## 3层诊断框架

### 第一层: 商业模式层

**核心问题** (5个必答):
1. 你的产品是什么? (必须可付款)
2. 谁会买? (具体人群,不是"所有人")
3. 为什么买? (真实需求,不是你的想象)
4. 如何卖? (具体渠道和流程)
5. 能赚多少? (具体数字,不是"应该不错")

**诊断标准**:
- ✅ 能在5分钟内说清楚商业模式
- ✅ 能立即发出付款链接
- ✅ 能说出至少3个已付款客户
- ❌ "我还在研究产品"
- ❌ "我的目标用户是所有想XX的人"
- ❌ "等我做出来就知道了"

**评分** (0-10分):
- 9-10分: 商业模式清晰,已有付费验证
- 7-8分: 商业模式基本清晰,需要验证
- 5-6分: 商业模式模糊,需要重新设计
- 3-4分: 商业模式不清晰
- 0-2分: 没有商业模式

**常见问题**:
| 问题 | 诊断 | 解决方案 |
|------|------|----------|
| "我的产品是内容" | ❌ 内容不是产品 | 设计可交易的产品(课程/咨询/工具) |
| "所有人都需要" | ❌ 目标过于宽泛 | 缩小到具体人群(如"AI开发者") |
| "应该能赚钱" | ❌ 没有验证 | 找到至少3个愿意付款的人 |
| "先做流量再变现" | ❌ 变现路径不清 | 先设计变现路径,再做流量 |

---

### 第二层: 执行层

**核心问题** (5个必答):
1. 你能不能搞定? (真实能力评估)
2. 你会不会去搞? (执行意愿)
3. 你搞了多少次? (实际行动次数)
4. 每次搞的结果如何? (数据追踪)
5. 下一步具体怎么搞? (明确计划)

**诊断标准**:
- ✅ 有SOP (标准操作流程)
- ✅ 有数据记录
- ✅ 有明确的下一步行动
- ❌ "我觉得我应该可以"
- ❌ "我打算开始做"
- ❌ "我做了几次,没什么效果"

**评分** (0-10分):
- 9-10分: 有SOP,有数据,执行100+次
- 7-8分: 有计划,有记录,执行10+次
- 5-6分: 有想法,少量执行
- 3-4分: 只有想法,没有执行
- 0-2分: 连想法都不清晰

**执行检查清单**:
```yaml
execution_checklist:
  - [ ] 有明确的SOP文档
  - [ ] 有数据追踪系统
  - [ ] 执行次数 ≥ 10次
  - [ ] 有对标对象
  - [ ] 有明确的下一步行动
  - [ ] 有时间表和里程碑
  - [ ] 有失败应对预案
```

**常见问题**:
| 问题 | 诊断 | 解决方案 |
|------|------|----------|
| "我应该可以" | ❌ 没有验证 | 先做10次,用数据说话 |
| "我打算做" | ❌ 没有行动 | 立即开始第1次 |
| "做了没效果" | ⚠️ 执行不足 | 至少执行100次再评估 |
| "不知道怎么做" | ❌ 缺少SOP | 找对标对象,复刻其SOP |

---

### 第三层: 认知层

**核心问题** (5个必答):
1. 你为什么要做这个? (真实动机)
2. 你相信这个能成吗? (信念强度)
3. 你愿意付出什么代价? (承诺程度)
4. 你害怕什么? (隐藏障碍)
5. 你在逃避什么? (深层问题)

**诊断标准**:
- ✅ 目标具体可量化
- ✅ 信念足够坚定
- ✅ 承诺足够明确
- ❌ "我想赚钱" (太模糊)
- ❌ "应该可以吧" (信念不足)
- ❌ "我不想太累" (承诺不足)

**评分** (0-10分):
- 9-10分: 目标清晰,信念坚定,承诺明确
- 7-8分: 目标基本清晰,有一定信念
- 5-6分: 目标模糊,信念不足
- 3-4分: 没有明确目标
- 0-2分: 自我欺骗,逃避现实

**认知检查清单**:
```yaml
cognitive_checklist:
  - [ ] 目标可量化 (如"6个月赚10万")
  - [ ] 信念坚定 (相信一定能成)
  - [ ] 承诺明确 (愿意每天投入X小时)
  - [ ] 无重大心理障碍
  - [ ] 愿意接受反馈
  - [ ] 不逃避困难
  - [ ] 不自我欺骗
```

**常见问题**:
| 问题 | 诊断 | 解决方案 |
|------|------|----------|
| "想赚钱" | ❌ 目标模糊 | 具体化: "6个月赚10万" |
| "应该可以" | ❌ 信念不足 | 找成功案例,建立信心 |
| "不想太累" | ❌ 承诺不足 | 重新评估是否真的想做 |
| "害怕失败" | ⚠️ 心理障碍 | 转到s-1.4-action-diagnostics |

## 诊断流程

### Step 1: 快速筛查 (5分钟)

**5分钟问答**:
1. 你的产品是什么?
2. 卖给谁?
3. 怎么卖?
4. 做了多少次?
5. 下一步做什么?

**筛查结果**:
- 能清晰回答 → 进入深度诊断
- 回答模糊 → 商业模式层问题
- 拒绝回答 → 认知层问题

---

### Step 2: 3层诊断 (30分钟)

**商业模式层诊断** (10分钟):
```
检查项:
- [ ] 产品可交易 (能发付款链接)
- [ ] 需求已验证 (有人付过款)
- [ ] 变现路径清晰 (不依赖单一收入)
- [ ] 定价合理 (基于市场而非成本)
- [ ] 可规模化 (不完全依赖个人时间)

输出:
- 商业模式可行 → 进入执行层
- 商业模式不清晰 → 输出设计方案
- 商业模式不可行 → 建议更换方向
```

**执行层诊断** (10分钟):
```
检查项:
- [ ] 有明确的SOP
- [ ] 有数据追踪系统
- [ ] 执行次数 ≥ 10次
- [ ] 有对标对象
- [ ] 有明确的下一步行动

输出:
- 执行到位 → 优化细节
- 执行不足 → 输出执行计划
- 执行偏差 → 对标校准
```

**认知层诊断** (10分钟):
```
检查项:
- [ ] 目标具体可量化
- [ ] 信念足够坚定
- [ ] 承诺足够明确
- [ ] 无重大心理障碍
- [ ] 愿意接受反馈

输出:
- 认知清晰 → 继续执行
- 认知模糊 → 目标澄清
- 认知障碍 → 转s-1.4-action-diagnostics
```

---

### Step 3: 问题消解 (10分钟)

**问题消解方法论**:
> "不要试图回答'我该怎么做',而是通过对标消解这个问题本身。"

**消解步骤**:
1. 识别问题类型 (商业/执行/认知)
2. 找到已解决该问题的对标对象
3. 拆解对标对象的解决方案
4. 像素级复刻其方法
5. 执行100次后评估

**示例**:
```
问题: "我不知道做什么内容"
传统方法: 思考、调研、分析...
消解方法: 找到成功的对标对象 → 复刻其内容类型和结构 → 执行100次

问题: "我的内容没人看"
传统方法: 优化标题、改进内容...
消解方法: 找到爆款内容 → 分析其成功要素 → 像素级复刻 → 执行100次

问题: "我不知道怎么变现"
传统方法: 研究变现模式、设计产品...
消解方法: 找到成功变现的对标 → 复刻其变现路径 → 执行100次
```
