## 增强概述

为 `a-1.2-deep-think` 增加**问题消解哲学**能力,基于维特根斯坦语言哲学、分析哲学和实用主义,帮助用户从根本上消解问题而非解决问题。

## 核心哲学框架

### 1. 问题消解 vs 问题解决

**核心区别**:
```
问题解决: 寻找答案
问题消解: 消解问题本身
```

**理论基础**:
- **Question** (疑问) → Answer (答案)
- **Problem** (麻烦) → Internal Struggle (内耗)
- 大多数人混淆了这两个概念
- 解决问题的前提是**定义问题**

**实践指导**:
- 不要急于回答"我该怎么做"
- 先问"这是一个 question 还是 problem"
- 如果是 problem,通过对标消解问题本身
- 如果是 question,通过追问逼近本质

### 2. 维特根斯坦语言哲学

**核心原理**:
```
语言的边界 = 思维的边界
```

**理论基础**:
- 使用垃圾语言就会拥有垃圾思维
- 认知升级的本质是**语言精确化**
- 没有本质定义,只有一系列相似特征的集合 (家族相似性)
- 词语的意义在于其在语言游戏中的使用

**语言游戏论**:
- 「确实」不是指向客观实在的词,而是在特定语言游戏中的一个棋子
- 应用案例: 什么是「道具」?
  - 不是本质定义
  - 而是多个维度的组合: 戏服、拍摄记录、公司保管、业务相关、发票注明等
- **这是消解"本质"问题的方法论**

**实践指导**:
1. 识别模糊语言 (如"确实"、"本质"、"真正的")
2. 用具体的、可操作的语言替代
3. 通过多维度描述替代单一本质定义

### 3. 分析哲学与逻辑哲学

**核心能力**:
```
说清楚 = 分析哲学 + 逻辑哲学
```

**理论基础**:
- 使用 Claude Code 的第一步不是"执行",而是"说清楚"
- "说清楚"可以筛掉 90% 的人
- 这是认知升级的关键瓶颈
- 第二步才是"执行" (涉及阿德勒心理学)

**说清楚的三个层次**:
1. **语义层**: 每个词的含义是什么
2. **逻辑层**: 论证结构是否有效
3. **实践层**: 如何转化为可执行的行动

**实践指导**:
- 不要使用模糊的、多义的词语
- 明确每个概念的边界
- 检查逻辑链条是否完整

### 4. 去魅化的实践理性

**核心立场**:
```
知识来自实践 (经验主义)
真理即有效 (实用主义)
拒绝本质 (反本质主义)
数据驱动 (实证主义)
去道德化 (价值中立)
```

**哲学谱系**:
```
洛克 (经验主义)
  ↓
休谟 (因果怀疑)
  ↓
杜威 (实用主义)
  ↓
维特根斯坦 (反本质主义)
```

**实践指导**:
- 不追求"本质"、"真理"
- 追求"有效"、"可行"
- 通过实践验证,而非理论推导
- 数据驱动决策,而非直觉

### 5. 逼近问题本质的方法

**核心方法**:
```
不断追问 → 逼近本质 → 拒绝回答错误的问题
```

**理论基础**:
- AI 的幻觉问题源于: 被设定为快速响应,即便用户提问模棱两可
- 解决方案: 通过追问消解错误的问题本身,而非回答错误的问题
- 创建一个 Agent,不断追问直到逼近问题的本质,否则拒绝回答

**追问的五个层次**:
1. **澄清层**: 你说的X是什么意思?
2. **假设层**: 你默认了什么前提?
3. **目标层**: 你真正想要的是什么?
4. **约束层**: 有哪些限制条件?
5. **本质层**: 这个问题的核心是什么?

**实践指导**:
- 不要急于回答
- 先追问 3-5 轮
- 直到问题清晰到可以用一句话描述

## 问题消解的三层框架

### 第一层: 语言精确化 (消解认知问题)

**目标**: 通过精确定义消解模糊问题

**方法**:
1. **区分 Question vs Problem**
   - Question: 需要答案的疑问
   - Problem: 需要消解的麻烦

2. **应用维特根斯坦语言游戏论**
   - 识别模糊词语
   - 用多维度描述替代本质定义
   - 明确语言游戏的规则

3. **避免垃圾语言**
   - 不使用"确实"、"本质"、"真正的"等模糊词
   - 用具体的、可操作的语言
   - 每个词都有明确的边界

**输出**:
```yaml
language_precision:
  original_question: <原始问题>
  question_type: question | problem
  ambiguous_terms: [<模糊词语列表>]
  precise_reformulation: <精确重新表述>
  operational_definition: <可操作定义>
```

### 第二层: 对标消解 (消解执行问题)

**目标**: 通过对标消解"我该怎么做"的问题

**方法**:
1. **找到对标对象**
   - 已经超额完成目标的人
   - 利润显著 (10倍以上)
   - 你能理解其模式
   - 你能复刻执行

2. **像素级复刻**
   - 不问"我该怎么做"
   - 而是"他怎么做的,我就怎么做"
   - 忽略现有资源/个人偏好
   - 执行而非过度思考

3. **消解问题本身**
   - 不是寻求答案
   - 而是通过对标消解问题
   - 从"如何做"变成"照着做"

**输出**:
```yaml
benchmark_dissolution:
  original_problem: <原始问题>
  benchmark_target: <对标对象>
  success_metrics: <成功指标>
  replication_plan: <复刻计划>
  dissolved: true  # 问题已消解
```

### 第三层: 心理诊断 (消解心理障碍)

**目标**: 通过诊断消解从"知"到"行"的障碍

**方法**:
1. **识别心理障碍**
   - 99% 的创业问题都是心理问题
   - 从知识层面,大家都知道
   - 从行动层面,真正愿意干的人并不多
   - 人们会为了让自己「不行」而刻意选择「不知」

2. **三层诊断**
   - **商业模式层**: 产品、客户、需求、渠道、收入
   - **执行层**: 能力、意愿、行动次数、数据、计划
   - **认知层**: 动机、信念、承诺、障碍、逃避

3. **消解障碍**
   - 不是"如何克服障碍"
   - 而是"识别障碍的本质,从而消解"
   - 从"我不知道怎么做"变成"我知道但不想做"
   - 从"我不想做"变成"我知道为什么不想做"

**输出**:
```yaml
psychological_dissolution:
  knowledge_level: <知识层面评估>
  action_level: <行动层面评估>
  gap_analysis: <知行差距分析>
  psychological_barriers: [<心理障碍列表>]
  dissolution_strategy: <消解策略>
```

## 问题消解的实战流程

### 流程图

```
用户输入问题
  ↓
第一步: 识别问题类型
  ├─ Question (疑问) → 需要答案
  └─ Problem (麻烦) → 需要消解
  ↓
第二步: 语言精确化
  ├─ 识别模糊词语
  ├─ 应用维特根斯坦分析
  └─ 重新表述问题
  ↓
第三步: 判断是否可对标消解
  ├─ 是 → 找对标对象 → 像素级复刻 → 问题消解
  └─ 否 → 继续下一步
  ↓
第四步: 心理诊断
  ├─ 识别知行差距
  ├─ 诊断心理障碍
  └─ 制定消解策略
  ↓
第五步: 综合评估
  ├─ 问题已消解 → 输出消解报告
  └─ 问题未消解 → 进入 P1-P4 深度分析
```

### 示例: "我该如何做好内容创作?"

**原始问题**:
```
"我该如何做好内容创作?"
```

**P0: 问题消解**

**第一步: 识别问题类型**
- 类型: Problem (麻烦)
- 原因: 这不是一个需要答案的疑问,而是一个需要消解的执行问题

**第二步: 语言精确化**
- 模糊词语: "做好"、"内容创作"
- 维特根斯坦分析:
  - "做好"没有本质定义,需要多维度描述: 播放量? 涨粉数? 变现额? 影响力?
  - "内容创作"是一个宽泛的语言游戏,需要明确: 短视频? 图文? 播客? 平台?
- 精确重新表述:
  ```
  "我想在抖音上做短视频,目标是3个月内涨粉10万,如何实现?"
  ```

**第三步: 对标消解**
- 是否可对标: 是
- 对标对象: 已经在抖音涨粉10万+的创作者
- 复刻计划:
  1. 找到 10 个对标账号
  2. 分析其选题、开头、节奏、标题
  3. 像素级复刻其模式
  4. 发布 100 条内容测试
- 问题消解: 是
  - 从"我该如何做"变成"他怎么做的,我就怎么做"

**第四步: 心理诊断**
- 知识层面: 已知道要对标
- 行动层面: 可能不愿意执行
- 知行差距: 担心"抄袭"、"没有创意"
- 心理障碍: 完美主义、自我怀疑
- 消解策略:
  - 认知重构: 对标不是抄袭,是学习
  - 行为实验: 先发 10 条测试
  - 小步快跑: 不追求完美,追求完成

**第五步: 综合评估**
- 问题已消解: 是
- 剩余问题: 无
- 输出: 消解报告 + 对标执行计划

## 追问引擎

### 追问的五个层次

**层次1: 澄清层**
- 目标: 明确词语的含义
- 问题:
  - "你说的X是什么意思?"
  - "X包括哪些具体内容?"
  - "X的边界在哪里?"

**层次2: 假设层**
- 目标: 识别隐含假设
- 问题:
  - "你默认了什么前提?"
  - "如果这个前提不成立会怎样?"
  - "为什么你认为X是理所当然的?"

**层次3: 目标层**
- 目标: 找到真正的目标
- 问题:
  - "你真正想要的是什么?"
  - "如果实现了X,你会得到什么?"
  - "X是目标还是手段?"

**层次4: 约束层**
- 目标: 明确限制条件
- 问题:
  - "有哪些限制条件?"
  - "什么是你不能改变的?"
  - "什么是你可以改变的?"

**层次5: 本质层**
- 目标: 逼近问题本质
- 问题:
  - "这个问题的核心是什么?"
  - "如果用一句话描述,你会怎么说?"
  - "去掉所有细节,剩下的是什么?"

### 追问示例

**用户**: "我想做自媒体赚钱"

**追问1 (澄清层)**: "你说的'自媒体'是指什么? 短视频、图文、播客,还是其他?"

**用户**: "短视频"

**追问2 (澄清层)**: "哪个平台? 抖音、视频号、B站?"

**用户**: "抖音"

**追问3 (目标层)**: "你说的'赚钱'是指多少? 月入1万、10万,还是其他?"

**用户**: "月入5万"

**追问4 (约束层)**: "你现在有多少粉丝? 每天能投入多少时间?"

**用户**: "0粉丝,每天2小时"

**追问5 (本质层)**: "所以你的问题是: 如何在抖音上,从0粉丝开始,每天投入2小时,在X个月内实现月入5万?"

**用户**: "对,就是这个问题"

**消解**:
- 问题类型: Problem (执行问题)
- 消解方法: 对标
- 对标对象: 已经实现月入5万的抖音创作者
- 复刻计划: 分析其选题、内容、变现模式,像素级复刻

## 常见问题类型与消解策略

### 类型1: 模糊目标型

**特征**: "我想做好X"、"如何提升X"
**问题**: 目标不明确,无法衡量
**消解策略**:
1. 追问具体指标
2. 明确时间范围
3. 转化为可衡量的目标

**示例**:
- 模糊: "我想做好内容"
- 精确: "我想在3个月内,在抖音上涨粉10万"

### 类型2: 本质追问型

**特征**: "X的本质是什么"、"什么是真正的X"
**问题**: 追求不存在的"本质"
**消解策略**:
1. 应用维特根斯坦家族相似性
2. 用多维度描述替代本质定义
3. 明确使用语境

**示例**:
- 错误: "什么是真正的好内容?"
- 正确: "在抖音平台,针对25-35岁女性用户,关于职场话题,播放量>10万的内容,通常具有以下特征..."

### 类型3: 执行障碍型

**特征**: "我知道要做X,但是..."
**问题**: 知行差距,心理障碍
**消解策略**:
1. 识别真实障碍 (不是"不知道",而是"不想做")
2. 诊断心理根源 (完美主义、畏难情绪等)
3. 制定消解策略 (认知重构、行为实验)

**示例**:
- 表面: "我不知道如何开始"
- 本质: "我知道如何开始,但害怕失败"
- 消解: 认知重构 + 小步快跑 (先发10条测试)

### 类型4: 对标可消解型

**特征**: "如何实现X"、"怎么做到X"
**问题**: 可以通过对标消解
**消解策略**:
1. 找到已实现X的对标对象
2. 分析其行为模式
3. 像素级复刻
4. 问题消解

**示例**:
- 问题: "如何在抖音涨粉10万?"
- 消解: 找10个已涨粉10万+的账号,分析其模式,复刻执行

### 类型5: 理论探索型

**特征**: 纯粹的知识探索,无执行意图
**问题**: 这是真正的 Question,需要答案
**消解策略**:
1. 不消解,进入 P1-P4 深度分析
2. 提供理论框架和多维度分析
3. 输出知识报告

**示例**:
- 问题: "AI对创意工作的影响是什么?"
- 策略: 进入 P1-P4 深度分析,不需要消解

## 输出示例

### 完整的 P0 问题消解报告

```markdown
# 问题消解报告

> 生成时间: 2026-04-30
> 原始问题: "我该如何做好内容创作?"

## 一、问题类型识别

**问题类型**: Problem (执行问题)
**判断依据**: 这不是一个需要理论答案的疑问,而是一个需要消解的执行障碍

## 二、语言精确化

### 2.1 模糊词语识别

| 模糊词语 | 问题 | 精确替代 |
|---------|------|---------|
| "做好" | 无法衡量 | "3个月内涨粉10万" |
| "内容创作" | 范围过宽 | "抖音短视频" |

### 2.2 维特根斯坦分析

**"做好"的家族相似性**:
- 维度1: 播放量 (>10万)
- 维度2: 涨粉数 (>10万)
- 维度3: 变现额 (月入5万)
- 维度4: 影响力 (行业认可)

**结论**: "做好"没有本质定义,需要明确具体维度

### 2.3 精确重新表述

**原始问题**:
```
"我该如何做好内容创作?"
```

**精确表述**:
```
"我想在抖音上做短视频,目标是3个月内涨粉10万,如何实现?"
```

## 三、对标消解

### 3.1 是否可对标

**判断**: 是
**理由**: 已有大量创作者实现了"3个月涨粉10万"的目标

### 3.2 对标对象

**对标标准**:
- 平台: 抖音
- 目标: 3个月涨粉10万+
- 领域: 与你相同或相近
- 可复刻: 你能理解并执行其模式

**对标账号** (示例):
1. @账号A - 职场领域,3个月涨粉15万
2. @账号B - 职场领域,3个月涨粉12万
3. @账号C - 职场领域,3个月涨粉20万

### 3.3 复刻计划

**第一步: 分析对标对象**
- 选题方向: 职场痛点、升职加薪、职场关系
- 开头模式: 递归式、数据冲击型
- 内容节奏: 15-30秒,快节奏
- 标题模板: 数字量化、对比反差

**第二步: 像素级复刻**
```
