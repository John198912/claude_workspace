## ⚠️ V2.0 方法论修正（必读，优先级高于下方旧文）

V1.0 的核心方法是"22 特征**全文盲扫**、打分越高越像 AI"。实践证明这套方法有根本缺陷：**全文盲扫会把作者的手迹当成瑕疵来扣分**——而冗余的语气词、忽长忽短的呼吸、不工整的对仗、口语化的插话，恰恰是"人写的"最强证据。把它们当 AI 痕迹删掉，结果是越查越没人味儿。这正是改稿环节最常见的失败模式。

V2.0 把本技能从"全文检测打分器"重构为"**改动段精准自查 + 人写迹象保护**"。下方 22 特征表保留，但**降级为信号参考库**，只在满足以下四条纪律的前提下使用：

### 纪律一：只审改动过/AI 生成的段落，不扫原文

检测范围只是"AI 生成的"或"这一轮被改动过的"句子，**不是全文**。作者亲笔写的原文不进入检测——它的毛边是手迹，不是检测目标。（与 `a-3.2.4-iterate` 模式 B 少动纪律配套：先判别稿件来源，模式 B 下只扫动过的句子。）

### 纪律二：人写迹象白名单（看到就保住，不准扣分）

以下都是**人写的证据**，检测时当作要保护的资产，绝不计为 AI 痕迹：
- 具体到难以编造的细节、带年代/圈子印记的梗
- 没解决的矛盾心情、自我打断和插话
- 长短不一的呼吸、书面语口语混用（这往往恰恰是真人）
- 作者能说出理由的用词选择、看似冗余的语气词（呢/吧/了，携带叹气、自嘲、犹豫）

### 纪律三：聚集判定（单个信号不算，多信号同现才是供词）

单个信号说明不了任何问题，以下都**不是** AI 的证据：语法完美/风格统一；一个破折号/一处排比/一个"然而"；干巴巴的文风。判断标准是**聚集**——一个破折号什么都不是；破折号 + 排比三连 + "璀璨的画卷" + "总而言之"段落同时出现，才构成 AI 味判定。

### 纪律四：不要误伤

报告里命中的孤立单信号一律不作为修改依据；只有聚集出现的信号簇才提出修改建议。修改建议针对的是"被改动段落里的 AI 痕迹"，不针对作者原文的个人风格。

### 六组 AI 味信号（聚集时才计）

下方 22 特征表是细化版；实操按这六组判断聚集即可：
1. **意义拔高**：把普通事实说成"标志着/见证了/折射出"；句尾挂补尾从句；拔到时代高度。
2. **宣传腔**："璀璨/深厚底蕴/赋能/打造/匠心/极致体验/不容错过"。
3. **句式套路**："不是 X 而是 Y"；排比三连；同义词轮换；假范围"从 X 到 Y"；格言公式；自问自答；假坦诚开场。
4. **格式痕迹**：破折号（最可靠 AI 信号）；加粗滥用；emoji 装饰；标题后重复标题的废话垫场。
5. **语气痕迹**：聊天残留（"希望这对你有帮助"）；谄媚（"好问题！"）；模糊归因（"有专家认为"）；万能展望结尾（"未来可期"）；签到式过渡（"让我们深入探讨"）。
6. **填充与对冲**："值得注意的是/需要指出的是"（删掉句子照常成立）；叠加对冲（"在某种程度上或许可能"）。

### 与质量门的接口

`content_quality_gate` 的改写回归检查据本修正层判定 AI 味：仅审改动过/AI 生成段落，命中孤立单信号不计为 AI 味；阈值沿用全系统口径（≥8 通过 / 7–7.9 条件通过+人工确认 / <7 退回修改）。

> 以下 V1.0 原文（22 特征表、检测算法、报告模板等）保留作为**信号细化参考**，但其"全文盲扫 + 越高越像 AI"的整体用法已被上方 V2.0 修正取代。阅读时以 V2.0 纪律为准。

## 技能定位

AI写作检测技能,通过22特征信号库识别**被改动段落/AI 生成段落**中的AI痕迹,生成质量报告,帮助创作者优化内容使其更具人性化和真实感。

**核心价值**:
- 提供22个典型AI特征的信号参考库（按 V2.0 纪律,仅用于改动段+聚集判定）
- 生成详细的质量评估报告（保护人写迹象,不误伤作者手迹）
- 提供具体的优化建议
- 确保内容通过平台审核

## 22特征扫描系统

### 类别1: 结构特征 (6项)

| 特征 | 描述 | 检测方法 | 权重 |
|------|------|----------|------|
| **过度结构化** | 过多的标题、列表、分段 | 统计标题/列表密度 | 高 |
| **完美对称** | 段落长度过于均匀 | 计算段落长度方差 | 中 |
| **模板化开头** | "在当今...""随着...的发展" | 关键词匹配 | 高 |
| **模板化结尾** | "总之...""综上所述..." | 关键词匹配 | 高 |
| **过度总结** | 频繁出现总结性段落 | 识别总结标记词 | 中 |
| **机械分点** | "首先...其次...最后..." | 序号词频率 | 中 |

### 类别2: 语言特征 (8项)

| 特征 | 描述 | 检测方法 | 权重 |
|------|------|----------|------|
| **词汇过于正式** | 缺少口语化表达 | 正式词汇比例 | 高 |
| **缺少个人化** | 无"我""你""咱们"等人称 | 人称代词统计 | 高 |
| **过度修饰** | 形容词和副词密度过高 | 词性分析 | 中 |
| **套话连篇** | "值得注意的是""需要强调" | 套话词库匹配 | 高 |
| **缺少情感** | 无感叹号、问号、省略号 | 标点符号分析 | 中 |
| **词汇重复** | 关键词过度重复 | TF-IDF分析 | 低 |
| **句式单一** | 句子结构过于相似 | 句法模式分析 | 中 |
| **缺少俚语** | 无网络用语、流行梗 | 俚语词库匹配 | 低 |

### 类别3: 内容特征 (5项)

| 特征 | 描述 | 检测方法 | 权重 |
|------|------|----------|------|
| **泛泛而谈** | 缺少具体案例和数据 | 具体性分析 | 高 |
| **观点中庸** | 避免极端观点和争议 | 情感极性分析 | 中 |
| **缺少细节** | 描述过于抽象 | 具体名词比例 | 高 |
| **过度全面** | 试图覆盖所有方面 | 主题分散度 | 中 |
| **缺少个人经验** | 无"我曾经""我发现" | 经验标记词 | 高 |

### 类别4: 风格特征 (3项)

| 特征 | 描述 | 检测方法 | 权重 |
|------|------|----------|------|
| **过于客观** | 缺少主观判断和态度 | 主观词汇比例 | 高 |
| **教科书式** | 像在讲课而非对话 | 教学标记词 | 中 |
| **缺少幽默** | 无调侃、自嘲、玩笑 | 幽默标记词 | 低 |

## 检测算法

### 算法流程

```python
def ai_check(content, threshold=6.0):
    """
    AI写作检测主函数

    Returns:
        score: 0-10分,分数越高越像AI
        features: 检测到的特征列表
        suggestions: 优化建议
    """

    # Step 1: 预处理
    text = preprocess(content)

    # Step 2: 22特征扫描
    feature_scores = {}
    for feature in FEATURES_22:
        feature_scores[feature] = check_feature(text, feature)

    # Step 3: 加权计算总分
    total_score = calculate_weighted_score(feature_scores)

    # Step 4: 生成报告
    report = generate_report(total_score, feature_scores)

    # Step 5: 生成建议
    suggestions = generate_suggestions(feature_scores)

    return {
        "score": total_score,
        "features": feature_scores,
        "suggestions": suggestions,
        "report": report
    }
```

### 评分标准

| 分数区间 | 判定 | 说明 |
|----------|------|------|
| 0-3分 | 人类写作 | 高度个性化,真实感强 |
| 3-5分 | 轻度AI痕迹 | 部分特征,可接受 |
| 5-7分 | 中度AI痕迹 | 明显特征,需优化 |
| 7-9分 | 重度AI痕迹 | 大量特征,强烈建议重写 |
| 9-10分 | 纯AI生成 | 几乎所有特征,必须重写 |

## 集成点

### M3 内容创作工作流
```
a-3.2.3-draft (初稿)
  → s-3.4.2-ai-check (AI检测)
  → a-3.2.4-iterate (迭代优化)
  → s-3.3-style-engine (风格校验)
  → s-3.4.1-fact-check (事实核查)
```

### 审核节点
- **自动触发**: 初稿完成后
- **阈值判断**: score > 6.0 → 阻止进入下一步
- **人工审核**: score 5.0-6.0 → 标记待审核
- **自动通过**: score < 5.0 → 进入下一步

## 限制与注意事项

### 限制
1. **语言**: 目前仅支持中文
2. **长度**: 建议500-10000字,过短或过长影响准确性
3. **领域**: 技术文档和学术论文可能误报率较高

### 注意事项
1. **不是绝对标准**: 分数仅供参考,最终判断需结合实际情况
2. **平台差异**: 不同平台对AI内容的容忍度不同
3. **持续优化**: 随着AI技术发展,检测标准需定期更新
