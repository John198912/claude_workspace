## 技能定位

内容诊断技能,在创作前通过5维度评估框架验证内容-市场契合度,识别潜在问题,提供优化建议,确保内容质量和传播效果。

**核心价值**:
- 创作前验证,避免无效产出
- 5维度全面评估
- 识别致命缺陷
- 提供可执行建议
- 提升内容成功率

---

## 调用方式

### Slash Command
```
/diagnose-content <topic_or_outline>
```

### @角色调用
```
@诊断师 评估这个选题的内容潜力
```

### 工作流集成
- `/create` 工作流: 大纲完成后,初稿前诊断
- `/quick-create` 工作流: 选题验证后诊断
- 手动调用: 任何需要评估的内容

## 输入要求

### 必需参数
- `topic`: 内容主题
- `outline`: 内容大纲 (可选,但建议提供)
- `target_audience`: 目标受众

### 可选参数
- `platform`: 目标平台 (默认: 全平台)
- `content_type`: 内容类型 (文章/视频/图文)
- `diagnostic_level`: 诊断级别 (quick/standard/thorough)

### 输入示例
```yaml
topic: "AI Agent开发完整指南"
outline: |
  1. AI Agent基础概念
  2. 开发环境搭建
  3. 核心功能实现
  4. 实战案例
  5. 部署与优化
target_audience: "AI开发者,技术爱好者"
platform: "公众号,知乎"
content_type: "长文"
diagnostic_level: "standard"
```

## 5维度诊断框架

### 维度1: 需求匹配度 (Demand Fit)

**评估问题**:
1. 目标受众是否真实存在?
2. 需求是否足够强烈?
3. 是否有人愿意为此付费/付出时间?
4. 竞品内容表现如何?
5. 搜索量和讨论热度如何?

**评分标准** (0-10分):
- 9-10分: 强需求,大量搜索和讨论
- 7-8分: 中等需求,有一定市场
- 5-6分: 弱需求,小众市场
- 3-4分: 需求不明确
- 0-2分: 伪需求,无市场

**诊断方法**:
- 搜索量分析 (百度指数/微信指数)
- 竞品内容数量和表现
- 社区讨论热度
- 付费意愿验证

**常见问题**:
- ❌ "所有人都需要" → 实际上没人需要
- ❌ "我觉得有用" → 不代表别人需要
- ❌ "朋友说不错" → 样本太小
- ✅ "搜索量1000+/天,竞品爆款多"

### 维度2: 差异化程度 (Differentiation)

**评估问题**:
1. 与现有内容有何不同?
2. 独特价值点是什么?
3. 为什么用户选择你而非竞品?
4. 差异化是否可持续?
5. 是否容易被模仿?

**评分标准** (0-10分):
- 9-10分: 独特视角/方法,难以复制
- 7-8分: 有明显差异,有一定壁垒
- 5-6分: 差异不明显,容易被超越
- 3-4分: 与竞品高度相似
- 0-2分: 完全同质化

**诊断方法**:
- 竞品内容对比分析
- 独特价值点提取
- 差异化可持续性评估
- 模仿难度评估

**常见问题**:
- ❌ "我写得更详细" → 不是差异化
- ❌ "我的案例不同" → 表面差异
- ❌ "我加了XX功能" → 容易被复制
- ✅ "我有独特方法论+实战数据+个人经历"

### 维度3: 可执行性 (Actionability)

**评估问题**:
1. 用户看完能立即行动吗?
2. 步骤是否清晰具体?
3. 是否提供工具/模板/资源?
4. 门槛是否合理?
5. 是否有验证反馈机制?

**评分标准** (0-10分):
- 9-10分: 步骤清晰,工具齐全,立即可用
- 7-8分: 步骤明确,需要一定准备
- 5-6分: 有指导,但不够具体
- 3-4分: 泛泛而谈,难以执行
- 0-2分: 纯理论,无法落地

**诊断方法**:
- 检查是否有SOP/清单/模板
- 评估步骤的具体程度
- 验证门槛的合理性
- 检查反馈机制

**常见问题**:
- ❌ "多学习,多实践" → 太模糊
- ❌ "根据自己情况调整" → 没有指导
- ❌ "需要XX年经验" → 门槛过高
- ✅ "3步操作+模板+案例+常见问题"

### 维度4: 传播潜力 (Virality)

**评估问题**:
1. 是否有传播钩子?
2. 是否引发情绪共鸣?
3. 是否有社交货币价值?
4. 标题是否吸引点击?
5. 是否适合平台算法?

**评分标准** (0-10分):
- 9-10分: 强传播钩子,高情绪价值
- 7-8分: 有传播点,中等吸引力
- 5-6分: 传播力一般
- 3-4分: 传播力弱
- 0-2分: 几乎无传播力

**诊断方法**:
- 钩子识别 (数字/对比/反常识/痛点)
- 情绪价值评估
- 社交货币测试 ("转发会让我显得XX")
- 标题吸引力测试
- 平台算法适配度

**常见问题**:
- ❌ "内容很好,自然会传播" → 酒香也怕巷子深
- ❌ "标题要严肃专业" → 可能无人点击
- ❌ "不迎合算法" → 可能无人看到
- ✅ "痛点+数字+对比+情感+平台优化"

### 维度5: 变现潜力 (Monetization)

**评估问题**:
1. 是否有明确的变现路径?
2. 用户付费意愿如何?
3. 客单价是否合理?
4. 转化流程是否顺畅?
5. 是否有后续产品?

**评分标准** (0-10分):
- 9-10分: 清晰变现路径,高付费意愿
- 7-8分: 有变现可能,需要设计
- 5-6分: 变现路径不清晰
- 3-4分: 变现困难
- 0-2分: 几乎无法变现

**诊断方法**:
- 变现路径设计
- 付费意愿调研
- 客单价对标
- 转化流程评估
- 产品矩阵规划

**常见问题**:
- ❌ "先做流量再考虑变现" → 可能永远不赚钱
- ❌ "靠广告赚钱" → 收入天花板低
- ❌ "产品还没想好" → 缺少规划
- ✅ "免费内容→低价产品→高价服务"

## 诊断流程

### Step 1: 快速筛查 (5分钟)
```
问题清单:
1. 目标受众是谁? (具体到人群画像)
2. 他们的核心痛点是什么?
3. 现有解决方案有哪些?
4. 你的独特价值是什么?
5. 用户看完能做什么?

快速判断:
- 5个问题都能清晰回答 → 进入深度诊断
- 3-4个能回答 → 需要补充信息
- ≤2个能回答 → 选题需要重新思考
```

### Step 2: 5维度评分 (15分钟)
```
每个维度:
1. 回答5个评估问题
2. 给出0-10分评分
3. 记录评分依据
4. 识别主要问题
5. 提出优化建议

总分计算:
- 需求匹配度 × 30%
- 差异化程度 × 25%
- 可执行性 × 20%
- 传播潜力 × 15%
- 变现潜力 × 10%
= 综合得分 (0-10分)
```

### Step 3: 问题诊断 (10分钟)
```
致命问题 (必须解决):
- 需求匹配度 < 5分
- 差异化程度 < 4分
- 可执行性 < 5分

严重问题 (强烈建议解决):
- 传播潜力 < 5分
- 变现潜力 < 4分
- 任何维度 < 6分

轻微问题 (建议优化):
- 任何维度 6-7分
```

### Step 4: 优化建议 (10分钟)
```
按优先级输出:
1. 致命问题的解决方案
2. 严重问题的优化建议
3. 轻微问题的改进方向
4. 整体优化策略
5. 下一步行动计划
```

## 集成点

### M2→M3 工作流
```
s-2.2-topic-validate (选题验证)
  → s-3.0-content-diagnostics (内容诊断)
  → a-3.2.2-outline (大纲设计)
```

### 审核节点
- **自动触发**: 大纲完成后,初稿前
- **阈值判断**:
  - 总分 ≥ 7.0 → 自动通过
  - 总分 5.0-6.9 → 标记待优化
  - 总分 < 5.0 → 阻止创作,建议重新选题
- **致命问题**: 任何致命问题 → 必须解决才能继续

## 配置参数

```yaml
content_diagnostics:
  enabled: true
  threshold: 7.0
  auto_block_fatal: true
  diagnostic_level: "standard"  # quick/standard/thorough

  # 维度权重
  dimension_weights:
    demand_fit: 0.30
    differentiation: 0.25
    actionability: 0.20
    virality: 0.15
    monetization: 0.10

  # 问题阈值
  thresholds:
    fatal: 5.0      # 低于此分数为致命问题
    serious: 6.0    # 低于此分数为严重问题
    minor: 7.0      # 低于此分数为轻微问题

  # 诊断级别
  levels:
    quick:
      time: 5min
      depth: "快速筛查"
    standard:
      time: 40min
      depth: "5维度完整诊断"
    thorough:
      time: 90min
      depth: "深度诊断+竞品分析"
```
