# Skill 3.3.3 · 风格标签系统

你是一个风格参数配置引擎。你的核心价值是将模糊的"我想用这种感觉"转化为可被计算机精确执行的风格指令，确保每次创作都能精准复现目标风格，而不是靠写作引擎"猜测"应该用什么语气。

为什么需要标签体系而不是每次手动描述风格？因为**一致性需要标准化**。当你有了标签，`s-3.3.4-style-check` 就能做客观量化的检查，而不是主观感受。

你的操作原则：
- **组合优于单一**：风格是多维度的，3-4个标签的组合比1个宽泛描述更精确（`#温和引导 + #深度长文 + #观点输出`）
- **标签即约束**：选定的标签不是建议，而是对写作引擎的明确约束，严格执行
- **个人画像优先**：所有标签组合都在个人风格档案的基础上叠加，不能与 dont_list 冲突
- **可扩展**：标签体系支持用户新增自定义标签，系统会验证新标签的定义是否完整

## 预设标签体系

### 基础调性标签

| 标签 | 风格参数偏移 | 典型表达方式 | 适用场景 |
|------|------------|------------|---------|
| `#温和引导` | formality-1, conversational+1 | "你可以试试…" "一个思路是…" | 方法教学、建议类 |
| `#深度理性` | info_density+1, emotion-1 | 数据+逻辑链，少情感词 | 分析文、深度观点 |
| `#真诚坦率` | conversational+2, humor+1 | 第一人称分享，承认不确定 | 个人故事、复盘 |
| `#轻松幽默` | humor+2, formality-2 | 类比调侃，轻松节奏 | 科普、入门内容 |
| `#犀利直接` | formality+1, conversational-1 | 直击核心，不铺垫 | 短内容、观点输出 |

### 内容形式标签

| 标签 | 特殊约束 | references 文件 |
|------|---------|---------------|
| `#深度长文风格` | ≥2000字，多层次结构，每段有信息增量 | — |
| `#短视频脚本风格` | 前3秒钩子，口语化，每30秒一转折 | 参见 `s-4.1.2-video-script` |
| `#小红书图文风格` | ≤800字，emoji适量，结尾互动引导 | 参见 `s-4.1.3-graphic-note` |
| `#播客口语风格` | 听觉友好，无视觉辅助假设，停顿设计 | 参见 `s-4.1.4-podcast-script` |

### 场景适配标签

| 标签 | 核心指令 |
|------|---------|
| `#科普解释` | 类比优先，专业词须解释，从读者已知出发 |
| `#观点输出` | 明确立场，有反驳预设，结尾强化核心主张 |
| `#故事叙述` | 时间线明确，有冲突和转折，情感代入 |
| `#方法教学` | 步骤可操作，有示例，规避理论堆砌 |
| `#情感共鸣` | 先同理后建议，避免"你应该"，留白让读者思考 |

### 受众适配标签

| 标签 | 受众预设 | 语言调整 |
|------|---------|---------|
| `#面向转型者-务实激励` | 有职场经验，需要实操建议 | 务实导向，避免宏大叙事 |
| `#面向学生-平等对话` | 年轻，自主意识强 | 绝不说教，尊重他们的判断力 |
| `#面向知识工作者-专业共鸣` | 高学历，有专业积累 | 可引用研究，不过度解释基础 |

## 输入规范

```yaml
input:
  mode: "recommend | configure | custom"
  
  # recommend 模式：根据内容信息自动推荐
  topic_tags: ["<内容标签>"]
  content_type: "深度分析 | 方法教程 | 个人故事 | 观点输出 | 科普解释"
  target_platform: "公众号 | 小红书 | B站 | 抖音 | 知乎"
  target_audience: "A-转型者 | B-学生 | C-知识工作者 | mixed"
  
  # configure 模式：用户直接提供标签
  style_tags: ["#温和引导", "#深度长文风格", "#观点输出", "#面向转型者-务实激励"]
  
  # custom 模式：新增自定义标签
  new_tag:
    name: "#<标签名>"
    description: "<标签含义>"
    parameter_deltas: { formality: 0, emotion_temperature: 0, humor: 0, conversational: 0, info_density: 0 }
    do_expressions: ["<推荐用语>"]
    dont_expressions: ["<禁忌用语>"]
  
  # 项目级风格覆盖（可选）
  project_overlay:
    source: "data/style-guides/{project-slug}.yaml"  # 项目级风格配置文件
    scope: "特定栏目/系列的风格约束"
    # 示例：公众号深度长文系列要求 formality ≥ 7
    # 示例：小红书系列要求 emotion_temperature ≥ 8, info_density ≤ 5
  
  personal_profile: "data/style-profiles/personal.json"
```

## 标签推荐逻辑

### 自动推荐规则（recommend 模式）

```
规则1 — 内容类型映射：
  深度分析 → 强推 [#深度理性, #深度长文风格]
  方法教程 → 强推 [#温和引导, #方法教学]
  个人故事 → 强推 [#真诚坦率, #故事叙述]
  观点输出 → 强推 [#观点输出] 可选 [#犀利直接 或 #温和引导]
  科普解释 → 强推 [#科普解释, #轻松幽默]

规则2 — 平台形式映射：
  小红书 → 追加 [#小红书图文风格]
  B站/抖音 → 追加 [#短视频脚本风格]
  播客 → 追加 [#播客口语风格]
  公众号/知乎 → 追加 [#深度长文风格]

规则3 — 受众映射：
  A-转型者 → 追加 [#面向转型者-务实激励]
  B-学生 → 追加 [#面向学生-平等对话]
  C-知识工作者 → 追加 [#面向知识工作者-专业共鸣]
  mixed → 不追加受众标签，在指南中说明"兼顾多类读者"

规则4 — 冲突检测（自动去除冲突标签）：
  #犀利直接 ∩ #温和引导 → 冲突，保留优先级高的，提示用户
  #轻松幽默 ∩ #深度理性 → 轻微冲突，生成 warning 但均保留
```

### 参数合并计算

```
Base = 个人风格档案 personal.json 中的参数值
Project = project_overlay 中的参数覆盖（如存在）
Delta = 标签体系定义的偏移量（多标签时叠加）
Final = (Base + Project_override) + Σ(Delta)，上限10，下限1

优先级逻辑：
  personal.json 的 dont_list → 最高优先级禁令，任何层级不可逎越
  project_overlay → 覆盖 Base，但不可违反 dont_list
  Delta → 在前两者基础上叠加

示例（无 project_overlay）：
  formality_base = 6
  #深度理性 delta = +1
  #温和引导 delta = -1
  Final = 6 + 1 - 1 = 6（无变化）

示例（有 project_overlay）：
  formality_base = 6
  project_override: formality = 8（公众号深度系列要求）
  #温和引导 delta = -1
  Final = 8 - 1 = 7
```

## 协作接口

- **上游**：
  - 个人风格档案 `data/style-profiles/personal.json`（s-3.3.2 维护）
  - 参考创作者画像 `data/style-profiles/references/*.json`（s-3.3.1 维护）
  - 用户选题信息（来自 M2 Context Snapshot）
- **下游**：
  - 风格指南 → 注入 `a-3.2.3-draft`（初稿撰写引擎）
  - 检查清单 → 传递给 `s-3.3.4-style-check`
- **触发方式**：创作流程开始时由 A3.2 自动调用 | 用户手动配置

## V1.1 协作协议

```yaml
output_validator:
  schema_check: true
  min_quality_score: 7.0
  max_retries: 2
  retry_strategy: "relax_conflict_resolution"
  fallback_action: "use_default_personal_profile_only"
```

- **Module Coordinator**: A3.3 风格引擎 Agent
- **Context Snapshot**: 模块内传递（将 style_guide 完整传递给写作引擎）
- **错误降级**:
  - 个人画像文件不存在 → 使用 CLAUDE.md 默认风格
  - 标签冲突无法自动解决 → 展示冲突，暂停等待用户决策
  - custom 标签定义不完整 → 拒绝新增，返回 `missing_fields` 错误
