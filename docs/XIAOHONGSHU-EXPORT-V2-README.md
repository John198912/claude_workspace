# 小红书内容导出助手 V2.0 使用说明书

## 📖 概述

**小红书内容导出助手** 是超级个体内容创作智能体群系统的配套工具，用于将 `/distribute` 命令生成的内容切片转换为小红书发布友好的格式。

### 为什么需要这个工具？

小红书平台有其独特的内容规范：
- 封面文案需在 20 字以内
- 正文最佳长度为 300-800 字
- 话题标签建议 10-15 个
- 发布时间影响流量（职场类早8点，成长类晚8点）

手动调整这些内容耗时且容易出错。本工具通过**智能分析、自动优化、一键导出**，大幅提升发布效率。

## ✨ 核心功能

### 1. 📊 智能字数检查

自动统计并检查以下内容：

| 检查项 | 限制 | 警告阈值 | 状态标识 |
|-------|------|---------|---------|
| 封面文案 | ≤20字 | >15字 | ✅/⚡/❌ |
| 正文内容 | 300-800字 | <200或>1000字 | ✅/⚡/❌ |
| 话题标签 | 10-15个 | <8或>20个 | ✅/⚡/❌ |

**状态说明**：
- ✅ 符合要求
- ⚡ 接近边界或略偏离最佳范围
- ❌ 超出限制或严重不足

### 2. 🤖 智能 Emoji 添加

根据正文内容自动添加匹配的 emoji：

| 关键词 | Emoji | 使用场景 |
|-------|-------|---------|
| 思考/为什么 | 🤔 | 提出问题时 |
| 问题 | ❓ | 疑问句 |
| 重要/注意 | ⚠️ | 强调重点 |
| 建议/方法 | ✅/📝 | 给出建议或步骤 |
| 结果/目标 | 🎯 | 总结成果 |
| AI/人工智能 | 🤖 | 提及AI相关内容 |
| 成长 | 🌱 | 个人成长话题 |
| 第一/1 | 1️⃣ | 步骤编号 |
| 第二/2 | 2️⃣ | 步骤编号 |
| ... | ... | ... |

**密度控制**：每 3-4 句话自动添加一个 emoji，避免过度使用。

### 3. 🧹 Markdown 清理

自动转换 Markdown 格式为小红书友好格式：

| 原始格式 | 处理后 | 说明 |
|---------|-------|------|
| `### 标题` | `标题` | 移除标题符号 |
| `- 列表项` | `列表项` | 移除列表符号 |
| `1. 步骤` | `步骤` | 移除序号 |
| `**加粗**` | `**加粗**` | ✅ 保留 |
| `` `代码` `` | `代码` | 移除代码符号 |

### 4. 💡 标签增强建议

当标签数量不足时，自动推荐补充标签：

**大话题**（通用流量）：
- #个人成长 #职场 #自我提升 #认知觉醒 #人生感悟

**中话题**（细分领域）：
- #AI时代 #超级个体 #一人公司 #职业转型 #副业赚钱

**小话题**（精准定位）：
- #内容创作 #写作技巧 #思维模型 #时间管理 #效率工具

### 5. ⏰ 智能发布时间建议

根据内容类型自动推荐最佳发布时间：

| 内容类型 | 最佳时间 | 原因 |
|---------|---------|------|
| 职场类 | 08:00 | 通勤时间，职场人群活跃 |
| 成长类 | 20:00 | 睡前阅读，成长焦虑高发 |
| 方法类 | 12:00 | 午休时间，适合收藏学习 |
| 其他 | 20:00 | 晚间黄金时段 |

### 6. 🔄 备选封面方案

自动生成 3 种不同风格的封面文案：

**方案A - 痛点型**
- 格式：`总觉得自己[关键词]不够？`
- 副标题：`可能问题不在你，在[关键词]`
- 适合：引发好奇心和共鸣

**方案B - 数字型**（如内容含数字）
- 格式：`[数字]个[关键词]真相`
- 副标题：`最后一个很多人不知道`
- 适合：收藏驱动

**方案C - 结论型**
- 格式：核心结论（前12字）
- 副标题：`看完这篇你就懂了`
- 适合：价值明确，深度内容

### 7. 📋 发布状态追踪

自动记录发布历史，防止重复发布：

```json
{
  "slice-001": {
    "date": "2026-03-15T23:19:18",
    "platform": "小红书",
    "published_at": "2026-03-15 23:19"
  }
}
```

## 🚀 快速开始

### 前置要求

- Python 3.7+
- 已执行 `/distribute` 命令生成切片内容

### 安装

无需安装，脚本已包含在项目 `scripts/` 目录下。

### 基本使用

#### 1. 单文件导出（查看）

```bash
python scripts/xiaohongshu-export.py "content/slices/20260307-001/slice-001-xiaohongshu-jinju.md"
```

输出到控制台，可直接复制使用。

#### 2. 单文件导出（保存）

```bash
python scripts/xiaohongshu-export.py "content/slices/20260307-001/slice-001-xiaohongshu-jinju.md" --output "./my-export"
```

保存到指定目录。

#### 3. 批量导出（推荐）

```bash
python scripts/xiaohongshu-export.py "content/slices/20260307-001" --batch
```

批量导出目录下所有 `*xiaohongshu*.md` 文件到 `publish-ready/` 目录。

#### 4. 指定内容ID导出

```bash
python scripts/xiaohongshu-export.py --content-id "20260307-001"
```

自动查找并导出该内容ID下的所有小红书切片。

#### 5. 记录发布历史

```bash
python scripts/xiaohongshu-export.py --record "slice-001"
```

标记该内容已发布，下次导出时显示"已发布"状态。

3. **选择封面方案**：从3个备选方案中选择最合适的

编辑 `scripts/xiaohongshu-export.py` 中的 `EMOJI_MAP`：

```python
EMOJI_MAP = {
    '你的关键词': '🔥',
    # ... 其他映射
}
```

### 自定义标签库

编辑 `POPULAR_TAGS` 添加你的常用标签：

```python
POPULAR_TAGS = {
    '大话题': ['你的大标签', ...],
    '中话题': ['你的中标签', ...],
    '小话题': ['你的小标签', ...],
```

修改 `XHS_LIMITS` 常量：

```python
XHS_LIMITS = {
    'cover_max': 20,        # 封面最大字数
    'body_optimal_min': 300, # 正文最佳最小值
    'body_optimal_max': 800, # 正文最佳最大值
    # ...
}
```

## 🔄 完整工作流

### 步骤 1：内容创作

```bash
/create "失业之后，我才开始真正工作"
```

生成母内容并完善。

### 步骤 2：内容分发

```bash
/distribute 20260307-001
```

生成多平台切片内容。

### 步骤 3：导出小红书

```bash
python scripts/xiaohongshu-export.py "content/slices/20260307-001" --batch
```

批量导出所有小红书内容。

### 步骤 4：制作图文

根据导出文件中的【设计指引】制作图片：
- 6-9 张 3:4 竖图
- 首图包含【封面文案】
- 风格统一，配色协调

### 步骤 5：发布内容

1. 打开小红书创作者后台
2. 上传图片
3. 复制【正文文案】到发布框
4. 复制【话题标签】到文末
5. 设定发布时间（参考【发布建议】）
6. 发布

### 步骤 6：记录发布

```bash
python scripts/xiaohongshu-export.py --record "slice-001"
```

记录发布历史，便于后续追踪。

## 🐛 故障排除

### 问题：Windows 控制台显示乱码

**解决方案**：
```bash
chcp 65001
python scripts/xiaohongshu-export.py ...
```

### 问题：找不到切片文件

**检查**：
1. 确认路径正确：`content/slices/{content-id}/`
2. 确认文件包含 `xiaohongshu` 字样
3. 确认文件扩展名为 `.md`

### 问题：字数统计不准确

**说明**：
- 封面字数：统计所有字符（含英文、数字）
- 正文字数：仅统计中文字符
- 标签数量：统计 `#` 分隔的标签数

### 问题：Emoji 添加不符合预期

**调整**：
修改 `add_smart_emoji()` 函数中的 `emoji_density` 参数：
- 值越小，emoji 越密集
- 值越大，emoji 越稀疏
- 建议范围：3-5
