# 小红书发布助手使用指南

## 功能概述

`xiaohongshu-export.py` 脚本将 `/distribute` 命令生成的切片内容转换为**小红书发布友好的格式**，支持一键复制粘贴到小红书创作者后台。

## V2.0 新增功能

- 📊 **字数统计面板**：实时显示封面/正文/标签字数，超限自动警告
- 🤖 **智能 Emoji 添加**：根据关键词自动添加 emoji（每3-4句一个）
- 🧹 **Markdown 清理**：自动清理 `###`、`- ` 等符号，保留 `**加粗**`
- 💡 **标签增强建议**：标签不足时自动推荐热门标签
- ⏰ **发布时间建议**：根据内容类型推荐最佳发布时间
- 🔄 **备选封面方案**：生成3种不同风格的封面文案供选择
- 📋 **发布状态追踪**：记录发布历史，防止重复发布

## 输出格式特点

- **分区明确**：封面文案、正文、话题标签、设计指引清晰分离
- **一键复制**：每个区域的内容可直接复制使用
- **发布清单**：内置检查清单，避免遗漏
- **批量处理**：支持一次性导出多个切片

## 使用方法

### 方式 1：单文件导出（打印到控制台）

```bash
python scripts/xiaohongshu-export.py "content/slices/20260307-001/slice-001-xiaohongshu-jinju.md"
```

直接查看格式化后的内容，复制所需部分。

### 方式 2：单文件导出（保存到文件）

```bash
python scripts/xiaohongshu-export.py "content/slices/20260307-001/slice-001-xiaohongshu-jinju.md" --output "导出目录"
```

### 方式 3：批量导出（推荐）

```bash
python scripts/xiaohongshu-export.py "content/slices/20260307-001" --batch --output "content/slices/20260307-001/publish-ready"
```

批量导出目录下所有小红书切片到 `publish-ready` 文件夹。

### 方式 4：通过内容ID生成完整发布包

```bash
python scripts/xiaohongshu-export.py --content-id "20260307-001"
```

### 方式 5：记录发布历史

发布完成后，记录到历史（防止重复发布）：

```bash
python scripts/xiaohongshu-export.py --record "slice-001"
```

## 发布流程

### Step 1：完成内容分发

```bash
/distribute 20260307-001
```

### Step 2：导出小红书内容

```bash
python scripts/xiaohongshu-export.py "content/slices/20260307-001" --batch
```

### Step 3：复制发布

打开 `publish-ready/slice-xxx-publish.txt` 文件：

1. **制作图片**：参考「设计指引」制作 6-9 张 3:4 竖图
2. **复制正文**：将「正文文案」部分复制到小红书发布框
3. **添加标签**：将「话题标签」复制到正文末尾
4. **检查清单**：确认所有检查项已完成

## 与 /distribute 命令的整合

可以在 `/distribute` 命令执行后自动调用导出脚本。

编辑 `.claude/commands/distribute.md`，在流程最后添加：

```yaml
[M4] 小红书内容导出（可选）
  └── scripts/xiaohongshu-export.py --content-id {content_id} --batch
      → 导出所有小红书切片到 publish-ready 目录
```

## 常见问题

**Q: Windows 控制台显示乱码？**
A: 脚本已内置编码修复，如遇问题可尝试：
```bash
chcp 65001
```

**Q: 如何自定义导出格式？**
A: 编辑 `scripts/xiaohongshu-export.py` 中的 `format_for_xiaohongshu()` 函数。

**Q: 支持其他平台吗？**
A: 当前仅优化小红书格式。如需其他平台（抖音、知乎等）的导出，可基于本脚本扩展。
