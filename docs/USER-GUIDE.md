## 1. 系统概览

本系统是面向"超级个体/一人公司"创作者的 AI 内容生产助手。通过 **5 个模块、16 个智能角色、~38 个 LLM Skill + 5 个脚本工具**，覆盖从知识积累到内容分发的完整链路。

### 系统定位

| 维度 | 说明 |
|------|------|
| **品牌方向** | 利用 AI 技术打造超级个体/一人公司 |
| **内容支柱** | 认知升级 · AI赋能 · 一人公司 · 个人成长 · 职业转型 |
| **核心风格** | 温和引导、深度分析、真诚分享——拒绝爹味与空洞鸡汤 |

### 五大模块

| 模块 | 职能 | 核心能力 |
|:----:|------|---------|
| **M1** | 知识积累 | 信息采集、深度思考、知识库建设 |
| **M2** | 选题策划 | 选题挖掘、评分验证、内容日历 |
| **M3** | 内容创作 | 体系设计、写作、风格校验、事实核查 |
| **M4** | 多平台分发 | 内容切片、平台适配、互动运营 |
| **M5** | 数据反馈 | 数据采集、内容复盘、策略迭代 |

## 2. 快速上手

### 2.1 第一次使用？从这里开始

```
步骤 1：学习一篇内容
  输入：/learn https://example.com/article

步骤 2：挖掘选题
  输入：/topic-mine

步骤 3：快速创作（推荐新手）
  输入：/quick-create 一人公司的核心竞争力

步骤 4：或完整创作（推荐深度内容）
  输入：/full-pipeline 一人公司的核心竞争力
```

### 2.2 我已经有思路了，想快速出稿

```
/quick-create <你的主题>
```

系统会走"素材研究→大纲→写作→风格快检→输出"的精简流程，**每步都会暂停等你确认**，预计 30-60 分钟完成。

### 2.3 我想做一篇高质量深度长文

```
/full-pipeline <你的主题>
```

系统会走完整的 M1→M5 全流程（选题验证→素材→大纲→写作→4轮迭代→风格校验→事实核查→SEO→多平台分发），预计 2-4 小时。

## 3. 三种调用方式

系统支持三种灵活度递增的调用方式，按需选择。

### 方式 1：/ 命令（全流程）

适合需要完整流水线的任务。输入斜杠命令触发预定义的多步流程。

```
/full-pipeline AI时代的个人品牌建设
/quick-create 为什么你应该成为超级个体
/learn https://example.com/article
/topic-mine --source knowledge
```

**何时用**：复杂任务、首次创作、需要完整质量保障时。

### 方式 2：@角色 点名（单一任务）

直接召唤某个特定角色完成单一任务，跳过调度开销。

```
@写手 基于以下大纲写一篇初稿：……
@风格师 检查这篇文章有没有爹味
@核查员 验证这篇文章里的数据是否准确
@适配器 把这篇长文改写成小红书图文
@研究员 帮我调研"一人公司"赛道的最新动态
@思考者 从创作者经济的角度分析这个观点
```

**何时用**：你清楚需要哪个角色、只需完成单一任务时。

### 方式 3：[任务:类型]（快速匹配）

按任务类型自动匹配最合适的 Agent 组合，最简调用。

```
[任务:研究] 调研 AI 写作工具的市场格局
[任务:选题] 给我挖掘 3 个本周可以写的选题
[任务:写作] 写一篇关于认知升级的文章
[任务:校验] 检查这篇文章的风格和数据
[任务:分发] 把这篇文章切片到小红书和抖音
[任务:复盘] 复盘上周发布的 3 篇内容
```

**何时用**：不想指定具体角色，让系统自动配置最佳团队时。

### 三种方式对比

| | / 命令 | @角色 | [任务:类型] |
|---|:---:|:---:|:---:|
| 流程控制 | 完整预定义流程 | 单步执行 | 自动匹配 |
| 灵活度 | 低（按流程走） | 高（精准指定） | 中（系统匹配） |
| 适合场景 | 复杂项目 | 明确单一任务 | 日常快速操作 |
| 学习成本 | 记住命令名 | 记住角色名 | 记住任务类型 |

## 4. 命令手册

### `/learn <input>` — 学习新内容

将外部信息学习并沉淀到知识库。

| 参数 | 说明 | 示例 |
|------|------|------|
| `<url>` | 从 URL 学习 | `/learn https://mp.weixin.qq.com/s/xxx` |
| `<关键词>` | 搜索后学习 | `/learn 超级个体 商业模式` |
| `--file <路径>` | 从本地文件 | `/learn --file docs/report.pdf` |
| `--quick` | 快速入库（跳过深度思考） | `/learn --quick https://...` |
| `--batch <urls>` | 批量学习多个 URL | `/learn --batch url1 url2 url3` |
| `--validate <id>` | 对已入库知识补充交叉验证 | `/learn --validate kb-001` |

**流程**：采集 → 摘要 → 深度思考（4轮） → 入库（标签+关联）

---

### `/topic-mine` — 挖掘选题

从知识库、趋势和受众痛点中发现候选选题。

| 参数 | 说明 |
|------|------|
| `--source knowledge` | 仅从知识库挖掘 |
| `--source trend` | 仅趋势热点 |
| `--source pain` | 仅受众痛点 |
| `--domain <领域>` | 限定特定领域 |
| `--platform <平台>` | 为特定平台优化 |
| `--quick` | 快速评分（跳过竞品分析） |

**输出**：按综合分排序的 TOP 10 选题清单，自动保存到 `data/topics.json`。

---

### `/validate <target>` — 质量验证

对选题、大纲或终稿进行多维度质量检查。

| 子命令 | 说明 |
|--------|------|
| `topic <描述/id>` | 验证选题可行性（评分+竞品+知识充分度） |
| `outline <文件>` | 验证大纲结构（完整性/逻辑/层次/受众匹配） |
| `content <文件>` | 终稿质量验证（风格+SEO+事实核查+Readiness评分） |
| `style <文件>` | 仅风格验证 |
| `knowledge <主题>` | 知识储备充分度检查 |

**输出**：问题清单 + 发布就绪度评分（0-100）。

---

### `/full-pipeline <topic>` — 完整创作流水线

从选题到多平台发布的一站式流程。

| 参数 | 说明 |
|------|------|
| `--topic-id <id>` | 使用已评分的选题库条目 |
| `--draft <file>` | 从已有初稿继续 |
| `--platforms <list>` | 指定目标平台 |
| `--style <tags>` | 指定风格标签组合 |

**完整流程**：
```
选题验证 → 素材补充 → 内容定位 → ⚠️ 确认
→ 大纲设计 → ⚠️ 确认 → 初稿撰写 → 4轮迭代
→ 风格校验 → ⚠️ 确认 → 事实核查 → SEO优化
→ 内容切片 → 多平台适配 → 发布清单 → ⚠️ 确认
```

**预计耗时**：2-4 小时 | **产出**：公众号长文 + 小红书图文 + 抖音脚本 + 播客脚本 + 发布清单

---

### `/quick-create <topic>` — 快速创作

精简版创作流程，**每步需人工确认**。

| 参数 | 说明 |
|------|------|
| `--platform <p>` | 指定目标平台（默认公众号长文） |
| `--style <tags>` | 指定风格标签 |
| `--skip-research` | 跳过素材研究 |
| `--adapt <platforms>` | 完成后自动适配其他平台 |

**流程**：
```
素材研究 → ⚠️ 确认 → 快速大纲 → ⚠️ 确认
→ 快速写作 → ⚠️ 确认 → 风格快检 → ⚠️ 确认
→ 输出
```

**预计耗时**：30-60 分钟

---

### `/distribute <content>` — 多平台分发

将完成的内容切片并适配多平台。

| 参数 | 说明 |
|------|------|
| `--file <path>` | 指定内容文件 |
| `--platforms <list>` | 指定平台 |
| `--slice-count <n>` | 切片数量目标（默认8） |
| `--skip-video` | 跳过视频脚本 |
| `--preview` | 仅预览切片计划 |

**支持平台**：公众号 / 小红书 / 抖音 / B站 / 知乎 / 即刻 / 播客

---

### `/review <period>` — 数据复盘

对已发布内容进行表现分析。

| 参数 | 说明 |
|------|------|
| `<content_id>` | 单篇复盘 |
| `--period monthly` | 月度综合复盘 |
| `--period quarterly` | 季度综合复盘 |
| `--all` | 批量复盘所有未复盘内容 |
| `--brand` | 个人品牌健康度报告 |

---

### `/style-learn <creator>` — 学习创作者风格

分析指定创作者的写作风格并建立风格档案，供后续创作参考。

## 6. 典型工作场景

### 场景 A：日常碎片化创作（30分钟）

> "我看到一个有趣的观点，想快速写一篇小红书帖子。"

```
1. /quick-create 为什么AI不会取代有深度思考能力的人 --platform 小红书
2. 每步确认 → 30分钟内完成
```

### 场景 B：深度长文创作（半天）

> "我想写一篇关于一人公司核心竞争力的深度文章，然后分发到多个平台。"

```
1. /learn 相关参考文章（先积累素材）
2. /full-pipeline 一人公司的核心竞争力
3. 按流程走完 → 自动生成多平台内容包
```

### 场景 C：已有初稿，需要打磨

> "我写了个草稿，想让系统帮我优化。"

```
1. @风格师 检查 content/drafts/my-draft.md
2. @核查员 验证这篇文章的所有数据
3. @适配器 把优化后的文章适配成小红书
```

### 场景 D：周末选题规划

> "周末空了，想规划下周的内容。"

```
1. /topic-mine（挖掘候选选题）
2. 挑选满意的选题
3. @日历官 把这3个选题排入下周日历
```

### 场景 E：月度复盘与策略调整

> "月底了，看看这个月内容表现怎么样。"

```
1. /review --period monthly
2. @策略师 根据复盘数据优化下月策略
```

## 7. 知识库操作

### 文件格式

每篇知识条目是一个带 YAML frontmatter 的 Markdown 文件。

```yaml
---
id: kb-20260307-001
title: "AI Agent 的协作架构设计"
date: 2026-03-07
tags: [AI, Agent, 架构]
category: technology
source: "https://example.com/article"
confidence_level: high    # high / medium / low
status: reviewed           # draft → reviewed → core
domain: AI技术
subdomain: Agent架构
content_type: 深度分析
audience: [觉醒者, 转型者]
creation_status: ready
content_form_fit: [深度长文, 视频脚本]
depth_level: deep
---

# 正文内容
...
```

### 存储路径

```
knowledge-base/
├── technology/        # 按 category 分类
├── mindset/
├── business/
├── growth/
└── viewpoint/
```

### 状态流转

```
draft（初次入库）→ reviewed（经过验证）→ core（核心知识）
```

使用 `/learn --validate <id>` 可以对已入库知识补充交叉验证，提升 confidence_level。

## 9. 人工审核节点

系统在关键节点会暂停，等你确认后才继续。标注 ⚠️ 的节点为必审。

### `/full-pipeline` 审核节点

| 节点 | 位置 | 说明 | 你会看到什么 |
|------|------|------|------------|
| 选题门控 | M2 结束 | 评分不达标时触发 | 评分详情 + 问题 |
| 定位确认 | M3 策略后 | 确认受众和角度 | 定位方案 |
| ⭐ 大纲审核 | M3 大纲后 | **最重要**，不通过不写稿 | 完整大纲 |
| 终稿审核 | M3 风格校验后 | 确认终稿质量 | 修改对照清单 |
| 事实核查 | M3 核查后 | 核查分不达标时触发 | 问题清单 |
| 发布确认 | M4 清单后 | 最终 sign-off | 各平台内容包 |

### `/quick-create` 审核节点

```
每一步完成后都会暂停：
  步骤1 素材研究 → ⚠️ 确认素材方向
  步骤2 快速大纲 → ⚠️ 确认大纲
  步骤3 快速写作 → ⚠️ 确认初稿
  步骤4 风格快检 → ⚠️ 确认终稿
```

每次确认时你可以选择：
- ✅ **确认继续**
- ✏️ **修改后继续**（说明修改点）
- 🔄 **重新执行本步骤**
- ❌ **终止**（已完成部分自动保存）

## 10. 错误处理与降级

系统在遇到问题时会自动降级处理，不会中断你的工作流。

### 常见降级方案

| 错误场景 | 系统会怎么做 |
|---------|------------|
| 知识库检索超时 | 改用 web-search 搜索替代 |
| 风格校验不合格 | 跳过自动改造，仅输出问题清单供人工处理 |
| 竞品分析无结果 | 使用历史缓存数据 |
| MCP 连接失败 | 提示你手动输入替代数据 |
| 事实核查不通过 | 输出问题清单，你确认修改后重新核查 |
| 某平台适配失败 | 跳过该平台，其他平台继续 |
| 内容过长（>10000字） | 分段处理，每段独立报告 |

### 数据安全

- 每步输出自动持久化到 `data/pipeline/{task_id}/`
- 支持断点恢复：使用 `/full-pipeline --draft <file>` 从中断处继续
- 错误日志记录到 `data/logs/error_log.jsonl`

## 附录：风格速查

### ✅ 推荐表达

```
"你可以试试……"
"一个思路是……"
"我的经验是……"
"换个角度看……"
"这里有个有意思的点……"
```

### ❌ 禁忌表达

```
"你应该……"     → 命令式，有爹味
"你必须……"     → 强制式
"年轻人就是要……" → 代际俯视
"相信我……"     → 空洞鸡汤
"我来教你……"   → 居高临下
```
