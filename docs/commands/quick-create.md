# `/quick-create <topic>` — 快速创作（每步人工确认）

> 平台中立命令定义 · 事实源：CLAUDE.md / CHANGELOG V1.2（V1.2 新增）
> 适配层：`.claude/commands/quick-create.md` 应为此文件的薄包装或符号链接

## 功能

轻量级创作模式，跳过完整 M1→M5 流程中的非必要环节，**每步完成后需人工确认**。预计 30-60 分钟（vs `/create` 的 2-4 小时）。

## 涉及模块

M3（精简流程）

## 品牌契约

本命令使用品牌定位信息。创作时须读取 `data/brand/brand-canon.yaml` 获取活跃品牌口径，并在输出中保持一致。品牌口径以 brand-canon.yaml 为准，不得使用已被取代的旧版品牌文案。

## 执行流程

```
步骤 1: 素材研究（精简） → ⚠️ 人工确认
步骤 2: 快速大纲（3-5 要点） → ⚠️ 人工确认
步骤 3: 快速写作 → ⚠️ 人工确认
步骤 4: 风格快检（仅禁忌扫描） → ⚠️ 人工确认
步骤 5: 输出
```

### 可选参数

- `--adapt <platforms>` — 输出时同时适配指定平台

### 与 `/create` 的区别

| 维度 | `/create` | `/quick-create` |
|------|-----------|-----------------|
| 预估耗时 | 2-4 小时 | 30-60 分钟 |
| 素材研究 | 完整 | 精简 |
| 迭代优化 | 3-5 轮 | 无/仅 1 轮 |
| 事实核查 | 完整 4 维度 | 无（仅风格快检） |
| SEO 优化 | 完整 | 跳过 |
| 人工确认 | 1 次（大纲） | 每步都确认 |
| 适用场景 | 深度长文 | 碎片化内容、热点、试验 |
