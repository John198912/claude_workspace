## 核心公理

> "执行力 + 商业模式 = 赚钱"

### 公理1: 问题识别的本质
"很多赚不到钱的人其实知道答案,只是在明知正解的情况下竭尽全力寻找绕过它的方法。"

**诊断要点**:
- 识别自欺欺人的逃避模式
- 区分"不知道"和"不想做"
- 暴露以"学习"为名的拖延

### 公理2: 产品的定义
"如果不能发付款链接让人成功付款,就是没有产品。"

**诊断要点**:
- 产品必须可交易
- 变现路径必须清晰
- 定价基于市场而非成本

### 公理3: 商业模式的本质
"供需匹配到极致、人货匹配到极致、交易摩擦缩小到极致。"

**诊断要点**:
- 需求是否真实存在
- 产品是否精准匹配
- 交易流程是否顺畅

## 三层诊断框架

### 第一层: 商业模式层

**诊断问题**:
1. 你的产品是什么? (必须可付款)
2. 谁会买? (具体人群,不是"所有人")
3. 为什么买? (真实需求,不是你的想象)
4. 如何卖? (具体渠道和流程)
5. 能赚多少? (具体数字,不是"应该不错")

**常见问题**:
- ❌ "我还在研究产品"
- ❌ "我的目标用户是所有想XX的人"
- ❌ "等我做出来就知道了"
- ✅ "我的产品是XX,售价XX,已有X人付款"

**诊断标准**:
- 能否在5分钟内说清楚商业模式
- 能否立即发出付款链接
- 能否说出至少3个已付款客户

### 第二层: 执行层

**诊断问题**:
1. 你能不能搞定? (真实能力评估)
2. 你会不会去搞? (执行意愿)
3. 你搞了多少次? (实际行动次数)
4. 每次搞的结果如何? (数据追踪)
5. 下一步具体怎么搞? (明确计划)

**常见问题**:
- ❌ "我觉得我应该可以"
- ❌ "我打算开始做"
- ❌ "我做了几次,没什么效果"
- ✅ "我已经做了50次,转化率3%,下一步优化XX"

**诊断标准**:
- 是否有SOP (标准操作流程)
- 是否有数据记录
- 是否有明确的下一步行动

### 第三层: 认知层

**诊断问题**:
1. 你为什么要做这个? (真实动机)
2. 你相信这个能成吗? (信念强度)
3. 你愿意付出什么代价? (承诺程度)
4. 你害怕什么? (隐藏障碍)
5. 你在逃避什么? (深层问题)

**常见问题**:
- ❌ "我想赚钱" (太模糊)
- ❌ "应该可以吧" (信念不足)
- ❌ "我不想太累" (承诺不足)
- ✅ "我要在6个月内赚到XX,愿意每天投入X小时"

**诊断标准**:
- 目标是否具体可量化
- 信念是否足够坚定
- 承诺是否足够明确

## 诊断流程

### Step 1: 快速筛查
**5分钟问答**:
1. 你的产品是什么?
2. 卖给谁?
3. 怎么卖?
4. 做了多少次?
5. 下一步做什么?

**筛查结果**:
- 能清晰回答 → 进入深度诊断
- 回答模糊 → 商业模式层问题
- 拒绝回答 → 认知层问题

### Step 2: 商业模式诊断
**核心检查**:
- [ ] 产品可交易 (能发付款链接)
- [ ] 需求已验证 (有人付过款)
- [ ] 变现路径清晰 (不依赖单一收入)
- [ ] 定价合理 (基于市场而非成本)
- [ ] 可规模化 (不完全依赖个人时间)

**诊断输出**:
- 商业模式可行 → 进入执行层诊断
- 商业模式不清晰 → 输出商业模式设计方案
- 商业模式不可行 → 建议更换方向

### Step 3: 执行层诊断
**核心检查**:
- [ ] 有明确的SOP
- [ ] 有数据追踪系统
- [ ] 执行次数 ≥ 100次
- [ ] 有对标对象
- [ ] 有明确的下一步行动

**诊断输出**:
- 执行到位 → 优化细节
- 执行不足 → 输出执行计划
- 执行偏差 → 对标校准

### Step 4: 认知层诊断
**核心检查**:
- [ ] 目标具体可量化
- [ ] 信念足够坚定
- [ ] 承诺足够明确
- [ ] 无重大心理障碍
- [ ] 愿意接受反馈

**诊断输出**:
- 认知清晰 → 继续执行
- 认知模糊 → 目标澄清
- 认知障碍 → 心理诊断 (转 action 框架)

## 诊断原则

### 原则1: 对标优于创新
"从0到1阶段,一定要找到对标对象。如果找不到完全一致的对标,问题通常是你的执行不到位,而非市场没有。"

### 原则2: 流量≠赚钱
"涨粉快往往意味着赚钱能力弱。真正的商业模式能在低流量下实现高转化。"

### 原则3: 商业模式是滤镜
"同样的认知水平,不同的商业模式导致完全不同的收入结果。"

### 原则4: 执行优于完美
"从0到1阶段,执行100次比完美计划更重要。"

### 原则5: 问题消解而非问题解决
"不要试图回答'我该怎么做',而是通过对标消解这个问题本身。"

## 常见诊断场景

### 场景1: "我不知道做什么"
**诊断**:
- 商业模式层: 没有明确产品
- 执行层: 没有尝试过任何方向
- 认知层: 害怕选择错误

**处方**:
1. 列出3个可能的方向
2. 每个方向找到对标对象
3. 选择对标对象最成功的方向
4. 执行100次后再评估

### 场景2: "我做了但没效果"
**诊断**:
- 商业模式层: 可能可行
- 执行层: 执行次数不足或偏差大
- 认知层: 期望过高或耐心不足

**处方**:
1. 检查执行次数 (是否 ≥ 100次)
2. 对比对标对象 (找出偏差)
3. 校准执行 (像素级复刻)
4. 继续执行至少100次

### 场景3: "我有流量但不赚钱"
**诊断**:
- 商业模式层: 变现路径不清晰
- 执行层: 可能执行到位
- 认知层: 混淆流量与商业模式

**处方**:
1. 重新设计变现路径
2. 找到高转化的对标对象
3. 复刻其变现方式
4. 优化转化流程

### 场景4: "我不想做低端的事"
**诊断**:
- 商业模式层: 可能清晰
- 执行层: 拒绝执行
- 认知层: 自我认知与现实脱节

**处方**:
1. 澄清"低端"的定义
2. 对比收入数据 (低端但赚钱 vs 高端但不赚钱)
3. 选择: 接受现实或更换方向
4. 如果选择接受,立即开始执行
