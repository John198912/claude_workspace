# 因为手段太多了，这本身就不适合用 single-agent 来解决问题，想做一个工具来容纳所有方法是不可能的

## 知识陈述

## 原始推文

> 当你做内容的方法足够多之后，会发现每一个环节，都需要 N 个智能体来支撑\n\n因为手段太多了，这本身就不适合用 single-agent 来解决问题，想做一个工具来容纳所有方法是不可能的\n\n而维护 100 个智能体的成本又太高\n\n放弃把所有的技巧都落地成工具了，就这么着吧
