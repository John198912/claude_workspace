## 与 v7 / v8 的差异（为什么需要 v9）

| 维度 | v7-merged | v8-implicit | **v9-ops** |
|------|-----------|-------------|-----------|
| 融合源 | v4 + v6 | 基于 v7 | **v2 + v4 + v6（三版）** |
| 人群区分 | 显式"如果你已经有稳定收入/还在找方向" | 隐性（分析段半显式，行动段隐性） | **隐性（分析段用"一种人/另一种人"平行陈述，行动段零分组）** |
| 行动段 | 两件分叉工具 + 一个共享动作 | 一个问题 + 一个动作（偏文学） | **一张纸·两栏·一个问题·定期重做（v2 的可操作清单回归，但通用化）** |
| 定位 | 公众号首发深度版 | 隐性叙事版 | **通用ops主版：最可部署、最落地** |

**核心增量**：v7/v8 在融合时丢掉了 v2 的「今晚就能做的三件事」可操作骨架，结尾偏向文学化（"在那里多待一会儿"）。v9 把 v2 的操作性脊柱接回来——但把"两栏清单 / 多付钱测试 / 定期复看"通用化，使同一套动作，已有事业者读作"收入结构审计"，还在探索者读作"探索复盘"，无需显式分组。这是 v9 之所以是 "ops"（可运营、可落地）版的原因。
