#### 3. s-3.0-content-diagnostics (内容诊断)

**功能**:
- 5维度诊断框架
  1. **需求匹配度** (30%权重): 目标受众、需求强度、市场验证
  2. **差异化程度** (25%权重): 独特价值、竞品对比、壁垒
  3. **可执行性** (20%权重): 步骤清晰、工具齐全、门槛合理
  4. **传播潜力** (15%权重): 传播钩子、情绪共鸣、平台适配
  5. **变现潜力** (10%权重): 变现路径、付费意愿、产品矩阵
- 0-10分综合评分
- 问题分级: 致命/严重/轻微
- 优化建议 (按优先级)
- 决策建议: proceed/revise/reject

**诊断流程**:
1. 快速筛查 (5分钟)
2. 5维度评分 (15分钟)
3. 问题诊断 (10分钟)
4. 优化建议 (10分钟)

**集成点**:
- M2→M3工作流: s-2.2-topic-validate → s-3.0-content-diagnostics → a-3.2.2-outline
- 阈值判断: 总分 < 5.0 → 阻止创作

**文件**: `skills/m3-creation/s-3.0-content-diagnostics/README.md`

  - L3: 你能复刻执行

- 对标对象分级: S/A/B/C级
- 像素级复刻计划生成
- 执行清单和时间表
- 成功指标设定

**对标方法论**:
- 对标优于创新 (0到1阶段)
- 像素级复刻 (不是"差不多")
- 执行100次后评估
- 问题消解而非问题解决
